Program 3: Synchronous vs Asynchronous

We could see in the last project program run in a step-step order or based on events.

That may not have been clear, so we will modify the project to be clearer.

When a computer does things in step-by-step order we called this synchronous.

The computer actually can do multiple things at once. Your code is written in a step-by-step synchronous manner.

When things happen in the computer outside of you code this is called Asynchronous.

When asynchronous things happen they can get back to you code by sending it an event.

The code you write synchrounously tells how to handle these events when they occur.

We'll rewite the last porgram to make this clearer.

Create the project

Create a new project like we did in program 1 except leave the file empty. Save it as app.js.

Add the code to input 

const readline = require('readline');
const input = readline.createInterface({
  input: process.stdin,
  output: process.stdout
});

Rewrite the code as follows:

process.stdout.write("One\n");

input.question("Hit Enter\n", (line)=>{
    process.stdout.write("Three\n")
    input.close(); 
})

process.stdout.write("Two\n");

What would you expect to see on the screen?

Remember, the program runs in a step-by-step order. The 'question' function is setting up how to handle an event, an event that hasn't occured yet.

Running the program

As described before in program 1

From the command line type: node app.js

C:\> node app.js

Notice it types out 

One
Hit Enter
Two

Not until you hit enter does it display "Three".

The your program is not doing anything after 'two' is printed. When the enter key is hit, the program is 'woken up' to respond to the event.

The code ran in order, but the 'question' function told the computer how to react to that event. It asked the question, but then went on to the next step.

Knowing when code runs in order or from reacting to events can confuse even the best of programmers. How the program runs is called 'The flow of execution'.

Many times its step-by-step. But external things can occur changing the flow of execution to go to a callback function.

Complete code for app.js

"use strict"

// Read line is a javascript object to accomplish:
const readline = require('readline');

// Readline can read from various inputs and outputs
// The following creates an instance of the interface using:
// keyboard input: stdin
// text output: stdout 
const input = readline.createInterface({
  input: process.stdin,
  output: process.stdout
});


// Let write to the screen
// the "\n" is a carriage return 
// which ends the line and move to the next line
process.stdout.write("One\n");

// Notice we need to wait to close the input
// until we are done using it
input.question("Hit Enter\n", (line)=>{
    process.stdout.write("Three\n")
    // This tells the system we are done using input and output
    input.close(); 
})

process.stdout.write("Two\n");
