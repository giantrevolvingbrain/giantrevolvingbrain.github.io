Program 2: Getting User Input

Computers will do the same thing over and over again unless external event cause it to do something else.

Users interacting with the computer create events that will alter the computer's actions.

In this program you will allow the computer to look for user input event and do something different.

Create the project

Create a new project like we did in program 1 except leave the file empty. Save it as app.js.

Using other people's work

Other people create code that is useful they place these into things called "Libraries" or "Modules".

This project will use a module called 'readline' that simplifies reading keyboard input. Readline can get the input one line at a time.

Using a module

Add this code to your app.js file.

const readline = require('readline');

The function 'require' will load the readline module into an object called 'readline'.

Using the object

The readline object allows you to create an iterface using the function 'createInterface' we called it 'input'.

Add the following to your app.js

const input = readline.createInterface({
  input: process.stdin,
  output: process.stdout
});

The iterface object uses two streams:

  • one for input (Think the keyboard) called stdin (Standard Input)
  • one for output (Think the screen) called stdout (Standard Output)

The output in this case is used to show the keys hit. Without it, when you hit a key nothing would be seen on the screen.

So when you hit a key "Y" it is an event sent to the input stream. The readline module automatically resends that to the output stream. The letter "Y" shows up on the screen.

Asking a question

The input object works with the input and output streams to put things on the screen and get input.

The input object has a function 'question'. Question outputs a questions and watches for events on the input stream. It keeps track of all the keys hit until the enter key is hit.

Once the enter key it sends an event with all the keys hit.

Add the following to your app.js 

input.question("Would you like to play a game?",  (line) => {
    process.stdout.write(line+'\n');
     // This tells the system we are done using input and output
    input.close(); 
})

This will ask the question "Would you like to play a game?". Separately the computer will now watch for the events described. When the enter key is hit, the function is called. This is called an event handler function and also a 'callback' function; it calls bacl to your program when an event occurs.

This syntax is a 'arrow function'.

 (line) => {
    . . .
 }

You also see the use of the ()=>{} syntax. Called 'fat arrow function' If you have used javascript before arrow is similar to function () {}

The arrow function is useful and makes clearer what is meant by the word 'this'.

'this' is a arrow means what you may expect: the game. 'this' in a function means the function, which is a bit confusing.

For more on Arrow function see here: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/Arrow_functions

When that event occurs the callback function will print out what was typed and close the input. 'close' tells it to stop looking for events.

If it is no longer looking for events, the program ends.

Running the program

As described before in program 1

From the command line type: node app.js

C:\> node app.js

You can type in anything then when you hit enter it prints out what was typed and the program ends.

Complete code for app.js

// Read line is a javascript object to accomplish user input
const readline = require('readline');

// Readline can read from various inputs and outputs
// The following creates an instance of the interface using:
// keyboard input: stdin
// text output: stdout 
const input = readline.createInterface({
  input: process.stdin,
  output: process.stdout
});


// Notice we need to wait to close the input
// until we are done using it
input.question("Would you like to play a game?",  (line) => {
   // Print the line sent to to the event handler functions
    process.stdout.write(line+'\n');
     // This tells the system we are done using input and output
    input.close(); 
})
