Program 4: Classes

Modules define reusable elements of code.

Classes allow you to group data and behavior that are related.

To learn more about classes see Documentation

We will create a module that has a single class to group the data and behavior for dealing with input and output with the screen.

We will call this the Terminal class. We will reuse this class in future projects.

Create the project

Create a new project like we did in program 1 except leave the file empty.

Save it as app.js.

Create another file and call it terminal.js

"use strict"

Constructor

The constructor is a method called when you create a 'new object' It is the place for adding values to the terminal.

This groups all the data we need for the class.

constructor() {
      this.output = process.stdout;
      this.input = readline.createInterface({
          input: process.stdin,
          output: process.stdout
      });
    }

The data values "output" and "input" are added to the object. The way that it happens is the use of the word 'this'

'this' can be read to 'to this object' do the following.

so 'this.output = process.stdout' adds the value "output" to this object and sets it the Standard Output.

Doing stuff to the terminal's output

Fill in the code for the write, writeLine and clearScreen function. These will all do stuff related to output.

   write(s) {
      this.output.write(s);
    }
    writeLine(s) {
      if (!s) s="";
      this.output.write(s+'\n');
    }
    clearScreen() {
        process.stdout.write('\x1Bc')
    }

The 'write' function outputs the passed value but stays on the current line.

The 'writeLine' function writes the passed value and then moves to the next line. It also looks for when no value is passed and makes sure a blank i.e. nothing but the new line is done.

The 'clearScreen' function clears the screen. This is just the command line screen.

The 'question' and 'close' functions are just like before, but here we are combining ito the Terminal class.

Using the terminal module

We now created a module, and now return to the app.js file so we can use it.

Add this code to the empty app.js

"use strict"

const Terminal = require('./terminal')
let terminal = new Terminal();

This should look similar to how we used readline. But, we add the './' in the require. This tells it to look in the same diretory as app.js. Also, you see the use of 'new' this is how classes create objects.

Using the terminal class

Lets finish the app.js by using some of the functions.

terminal.clearScreen();

terminal.question("Enter some text and Hit Enter\n", (line)=>{
    terminal.writeLine(line)
    terminal.close(); 
})

This will clear the screen and then Ask a question. This should seem familar, but now we have a reusable terminal. The program you write can reuse the code from Terminal making your code smaller.

"use strict"

const Terminal = require('./terminal')
let terminal = new Terminal();

// We can clear the screen
terminal.clearScreen();

// Notice we need to wait to close the input
// until we are done using it
terminal.question("Enter some text and Hit Enter\n", (line)=>{
    terminal.writeLine(line)
    // This tells the system we are done using input and output
    terminal.close(); 
})

Complete code for terminal.js

"use strict"
const readline = require('readline')
module.exports = class Terminal {
    constructor() {
      this.output = process.stdout;
      this.input = readline.createInterface({
          input: process.stdin,
          output: process.stdout
      });
    }
    write(s) {
      this.output.write(s);
    }
    writeLine(s) {
      if (!s) s="";
      this.output.write(s+'\n');
    }
    clearScreen() {
        process.stdout.write('\x1Bc')
    }
    question(prompt, callback) {
      this.input.question(prompt, callback);
    }
    close() {
      this.input.close();
    }
}
